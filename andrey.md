Гайд по разметке results.json
Что нужно сделать
У тебя есть файл results.json — это результаты эксперимента. GigaChat отвечал на 32 вопроса про ОмГТУ в двух режимах:

baseline — без базы знаний, просто GigaChat
rag — GigaChat + наша база данных ОмГТУ

Твоя задача — оценить каждый ответ по трём критериям. Это займёт 20-30 минут.

Как открыть файл
Открой results.json в любом редакторе. Рекомендую VS Code — он подсвечивает JSON и удобно сворачивает блоки.
Структура одного вопроса выглядит так:
json{
  "id": 1,
  "question": "Как получить академический отпуск в ОмГТУ?",
  "baseline": {
    "answer": "Для получения академического отпуска...",
    "time_sec": 1.68,
    "manual_score": null,
    "hallucination": null,
    "relevance": null
  },
  "rag": {
    "answer": "В ОмГТУ академический отпуск предоставляется...",
    "time_sec": 0.52,
    "manual_score": null,
    "hallucination": null,
    "relevance": null
  }
}
```

Тебе нужно заменить все `null` на `0` или `1`.

---

## Три критерия оценки

### 1. `manual_score` — правильность ответа
> Ответ содержит верную конкретную информацию про ОмГТУ?

- `1` — да, ответ правильный и полезный
- `0` — ответ неправильный, придуманный, или "не знаю"

**Примеры:**
```
Вопрос: "Кто декан ФТНГ?"
Baseline: "Деканом может быть назначен профессор кафедры..." → 0 (общий ответ, нет имени)
RAG: "Декан ФТНГ — Квасов И.Н." → 1 (конкретный правильный ответ)
```

---

### 2. `hallucination` — галлюцинация
> Модель выдумала факты которых нет или которые неверны?

- `1` — да, есть выдуманные факты (имена, адреса, даты, телефоны)
- `0` — нет галлюцинаций (ответ либо правильный, либо честный отказ)

**Примеры:**
```
Baseline: "Деканат ФИТиКС находится по адресу Мира 5, каб. 301" → 1 (адрес выдуман)
RAG: "В контексте не указано расположение деканата" → 0 (честный отказ, не галлюцинация)
```

---

### 3. `relevance` — релевантность
> Ответ вообще по теме вопроса?

- `1` — да, ответ про то о чём спрашивали
- `0` — ответ ушёл в сторону или про другой вуз

**Примеры:**
```
Вопрос: "Как добраться до 6 корпуса ОмГТУ?"
Baseline: "Чтобы добраться до корпуса МГУ..." → 0 (про другой вуз)
RAG: "От главного корпуса на троллейбусе №4..." → 1 (по теме)

Как заполнять
Найди в файле "manual_score": null и замени null на 0 или 1. То же самое для hallucination и relevance.
До:
json"baseline": {
  "answer": "Текст ответа...",
  "time_sec": 1.68,
  "manual_score": null,
  "hallucination": null,
  "relevance": null
}
После:
json"baseline": {
  "answer": "Текст ответа...",
  "time_sec": 1.68,
  "manual_score": 0,
  "hallucination": 1,
  "relevance": 1
}

Важные моменты
Оценивай независимо — сначала читай вопрос, потом ответ baseline, потом ответ rag. Не сравнивай их между собой, оценивай каждый отдельно.
Короткий правильный ответ — это 1, не 0. Если RAG ответил "Декан РТФ — Иванов А.В." — это отличный ответ, ставь 1.
Честный отказ — не галлюцинация. Если модель написала "В контексте нет информации" — hallucination: 0. Галлюцинация это когда модель придумала факт.
Если не уверен — ставь 0. Лучше занизить чем завысить.

После разметки
Сохрани файл и запусти в терминале из папки проекта:
bashpython score.py
Увидишь итоговую таблицу метрик. Скинь результат и сам файл обратно.
